# “我的”页面客服行与 Hero 统计面板修复设计

## 背景与根因

“反馈与建议”是微信原生 `<button open-type="contact">`。当前只重置了按钮边框、背景和内边距，没有在按钮自身明确恢复 `.setting-row` 所需的 flex 布局；在真机上，原生按钮的块级布局导致描述换行，箭头跟随文字而不是固定在行尾。

Hero 右侧“手数”“战绩”统计块被 P5 页面级覆盖统一设为 `var(--profile-panel)`，实际呈现为近黑色，与红色 Hero 的整体色彩割裂。

## 目标

- “反馈与建议”与同模块其他命令行使用同样的横向布局、间距和最右箭头。
- 正常手机宽度下，反馈描述保持一行；仅在真正不足的窄屏上自然换行，不使用强制 `white-space: nowrap`。
- Hero 两个统计块改为两级半透明深酒红背景，不再使用黑色面板。
- 保留黄色手数信息、绿色盈利信息的可读性和当前 Hero 斜切结构。

## 设计

### 客服反馈行

- `.customer-feedback-button` 明确设置 `display: flex`、`align-items: center`、`justify-content: space-between` 和与 `.setting-row` 相同的间距。
- `.setting-main` 继续承担可伸缩文字区域。
- `.setting-arrow` 增加 `flex-shrink: 0`，保证所有列表箭头尺寸稳定。
- 客服按钮内的箭头增加 `margin-left: auto`，确保它固定在最右侧。
- 不改变 `<button open-type="contact">`、文案、事件或客服行为。

### Hero 统计面板

- “手数”面板使用较亮的深酒红渐变：`rgba(112, 20, 38, 0.88)` 到 `rgba(72, 8, 24, 0.84)`。
- “战绩”面板使用稍深、稍偏紫的酒红渐变：`rgba(82, 14, 42, 0.88)` 到 `rgba(52, 7, 28, 0.84)`。
- 两个面板使用同一条低透明白色左边界，替代与红色背景混在一起的纯红边界。
- 保留现有文字颜色、装饰层、尺寸和布局，不重新设计 Hero。

## 非目标

- 不修改帮助与反馈的其他两行。
- 不修改其他命令列表、箭头字符、模块背景或底部导航。
- 不修改版本、路由、后端、客服配置和数据模型。
- 不上传微信开发版。

## 验收标准

1. 客服按钮本体明确为 flex 命令行，文字区域和箭头处于同一横向布局。
2. 客服箭头固定在最右侧，尺寸与新手引导、版本更新箭头一致。
3. 正常手机宽度下“反馈问题、Bug 或功能建议”不换行；窄屏仍允许自然换行。
4. 手数和战绩面板均不使用 `var(--profile-panel)` 或黑色背景。
5. 两个统计面板分别使用批准的两级酒红渐变，黄色和绿色数字保持可读。
6. 原生客服入口、Hero 数据绑定及其他 Profile 行为不变。
7. Profile 专项测试和完整测试通过。
8. 微信真实主工作区 `auto-preview` 成功，并完成同状态截图对照。

## 测试策略

- 在客服入口测试中精确断言按钮的 flex、对齐、间距和箭头右对齐规则。
- 在 Profile 视觉契约测试中精确断言两个 Hero 统计面板的渐变值，并禁止页面级统计块回退到 `var(--profile-panel)`。
- 运行 Profile 相邻回归、完整已提交测试和微信 `auto-preview`。
- 对照用户提供的两张截图检查反馈行、箭头和 Hero 面板；若模拟器无法复现登录态，则把手机截图确认列为唯一剩余验收项。
