# 智牌屋正式上线发布前准备与验收清单

## 1. 当前结论

截至 2026-07-22，当前代码候选版本为 `3.37`，已上传微信开发版本，构建号为
`1.0.20260722.2220.v3.37`。代码自动化、干净包预览、社交云函数部署和云端基础设施预检均通过，但**当前仍不建议立即提交正式审核**。

上线前仍有 4 类阻塞项：

1. 候选源码已经推送并创建标签，但回滚负责人尚未明确。
2. 广场文字评论、回复及公开昵称的服务端自动检测已部署，仍需在体验环境完成正常文字、违规文字和接口异常的真机验证；管理员事后删除继续作为人工兜底。本阶段不建设用户举报队列。
3. 好友邀请小程序码当前使用 `trial` 环境；正式发布后必须切换为 `release`，否则普通用户扫描新二维码可能仍被带到体验版。
4. 微信公众平台中的备案、服务类目、用户隐私保护指引和客服能力尚需人工逐项确认，不能由本地代码测试替代。

满足本文 P0 条件后，才进入“提交审核”；审核通过后，还要完成发布日切换和真机回归，才能正式对外宣传。

### P0-2 公开 UGC 合规能力

当前广场允许所有人浏览、点赞和评论，因此不能继续按“纯本地牌谱工具”处理。

最低上线方案：

- [x] 广场文字评论和回复接入服务端文本安全检测；明确违规、疑似需复核或检测服务异常时均不写入评论。
- [x] 只把用户实际输入的广场评论文字和公开昵称送检；好友/指定好友评论、内置扑克贴纸、牌谱、Note、Leak 和本地好友备注均不送检。
- [x] 保留管理员软删除评论能力，作为漏检内容的人工处置兜底。
- [x] 部署 `poker_social` 及 `security.msgSecCheck` 权限配置。
- [ ] 用真机分别验证正常文字、违规文字和接口异常提示，确认微信侧接口权限真实生效。
- [x] 首次微信昵称输入和后续资料编辑均使用原生昵称审核控件；公开昵称首次建立或变化时还会通过云端资料场景检测，失败时保留旧的公开昵称。
- [x] 牌谱本体继续只允许结构化牌谱，不允许自由文本正文；这部分不做通用内容审核。

当前边界：**检测广场文字评论、回复和公开昵称；不扫描好友/指定好友评论、私人 Note、Leak、本地好友备注、牌谱行动或私有手牌数据。** 举报队列、用户禁言和更完整的风控后台列为可拓展功能，不作为本阶段实现内容。

验收标准：违规或无法完成检测的广场文字无法公开；管理员可移除漏检评论；所有失败均不会泄露 OpenID、检测明细或私有牌谱。

### P0-3 审核定位与博彩误判风险

产品实际定位必须是“德州扑克牌谱记录与复盘训练工具”，不能表现为下注、结算、充值、提现、组织牌局或博彩引流服务。

提交前必须人工复查：

- [x] 将 Session 的“提现”改为“结束筹码”。
- [x] 首次体验和审核路径不再使用“现金局教练”“澳门 300/600”等高风险营销式文案。
- [x] 在统计和个人页说明结果来自用户自行录入，仅用于个人复盘，不代表真实资产或平台结算。
- [x] 代码扫描确认应用内不存在充值、提现、赛事报名、俱乐部招募、联系方式引流或博彩引流入口；本产品不提供下注、撮合或输赢排行榜。
- [x] 广场分享快照白名单及安全测试确认不返回金额、盈利、场所或真实玩家身份，只展示 BB 化匿名牌谱。

建议审核备注：

> 智牌屋是德州扑克牌谱记录与复盘训练工具。用户仅记录自己的线下学习数据，产品不提供扑克游戏、在线对局、下注、充值、提现、资金结算、赛事组织、俱乐部招募或博彩引流。公开分享仅包含 BB 化、匿名化的结构化牌谱，不展示真实金额、输赢或其他玩家身份。

验收标准：审核员从首次登录完成一轮核心体验时，不会看到可操作的资金功能、博彩宣传或陌生人牌局招募。

## 5. 审核、发布和发布后顺序

### 阶段 A：提交审核

1. 完成全部 P0 项并重新生成体验版。
2. 用全新微信账号走一遍首次登录、创建场次、记录手牌、查看统计、添加好友和发布牌谱。
3. 填写服务类目、页面路径、功能说明和上述审核备注。
4. 提交审核；不要在审核期间修改体验版或云函数行为。

### 阶段 B：审核通过后的正式发布

1. 确认审核通过版本与 Git 发布标签、体验版构建号完全一致。
2. 发布小程序正式版。
3. 将 `poker_social` 的 `SOCIAL_INVITE_QR_ENV_VERSION` 从 `trial` 切换为 `release`；只更新函数配置，不轮换邀请密钥、不修改管理员名单。
4. 用一个从未加入体验名单的普通微信扫描新生成的小程序码，确认进入正式版邀请页。
5. 验证微信分享卡片、搜索进入、冷启动和老用户升级。

### 阶段 C：上线后 24/72 小时

- 前 2 小时：每 15 分钟检查登录、云函数错误率、广场读取、评论和新增账号。
- 前 24 小时：每 2 小时检查一次，重点关注二维码、账号同步、缓存错乱、权限越权和 AI 超时。
- 24—72 小时：每天至少两次检查错误日志、管理员评论处置记录、云资源消耗和用户反馈。
- 出现跨账号数据、未匿名手牌、越权阅读或无法清除数据时，立即停止公开入口并回滚，不等待普通热修复。

## 6. 回滚标准

满足任一条件立即回滚或关闭对应能力：

- 不同微信账号看到彼此私有牌谱、Note、Leak、盈利或真实玩家名。
- 广场分享未完成 BB 化或非 Hero 玩家未匿名。
- 好友解除或范围变更后旧权限仍有效。
- 评论/昵称出现明显违规内容且无法及时处置。
- 云函数连续 5 分钟错误率超过 5%，或核心数据写入出现重复/丢失。
- 账号清除无法完成或清除后仍能恢复公开社交资料。

回滚顺序：先关闭广场/评论等可独立停用的高风险能力，再恢复上一稳定代码版本；数据库集合和索引不得自动删除。管理员配置异常时使用既有 fail-close 流程移除管理员能力。

## 7. 最终 Go / No-Go 门槛

只有同时满足以下条件才为 **GO**：

- P0 全部完成；
- `1075/1075` 或更多测试通过；
- 当前体验版对应唯一 Git 提交与发布标签；
- 三账号真机矩阵通过；
- 备案、类目、隐私指引和客服入口人工确认完成；
- 数据备份、告警、回滚负责人和发布后值守已经确定；
- 审核通过后已把邀请二维码环境切换到 `release` 并由非体验成员验证。

否则结论为 **NO-GO**，继续保留体验版，不进行正式发布。
