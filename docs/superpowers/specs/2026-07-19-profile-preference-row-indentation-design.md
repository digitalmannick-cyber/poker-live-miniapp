# Profile 偏好设置圆点与缩进设计

## 目标

让“偏好设置”模块的四个区块不再贴住列表左边缘，并复用“数据管理”等命令列表的灰色圆点语言，使整页左侧节奏一致。

## 已确认方案

采用方案 A：筹码单位、场地预设、级别预设、对手类型四个区块都增加圆点；每个区块内的标题、说明、筹码单位按钮和预设标签整体右移。编辑按钮继续保留当前右侧位置和现有点击热区。

## 结构

- 给偏好设置的 `profile-command-list` 增加独立类 `profile-preference-list`，避免影响数据管理、AI 自动提醒、帮助与反馈、账号与安全。
- 保留四个 `.setting-block` 的现有层级、事件、循环数据和按钮结构。
- 圆点由 `.setting-block::before` 绘制，不新增无语义 WXML 节点。

## 视觉规则

- 每个偏好区块左侧内边距调整为 `54rpx`，右侧保持现状。
- 圆点位于区块左侧 `22rpx`，尺寸 `10rpx × 10rpx`。
- 圆点颜色与命令列表一致：`rgba(255,255,255,0.22)`。
- 圆点光晕与命令列表一致：`0 0 0 8rpx rgba(255,255,255,0.03)`。
- 圆点与每个区块标题的第一行垂直对齐。
- 分隔线继续贯穿列表宽度，不随内容缩进变短。
- 不改变筹码单位按钮、预设标签和编辑按钮本身的视觉尺寸。

## 行为与范围

- `selectChipUnit`、`editVenues`、`editBlindPresets`、`editOpponentTypes` 行为保持不变。
- 88rpx 点击热区保持不变。
- 不修改其他 Profile 模块、版本号、路由、数据结构或后端。
- 不上传微信开发版本，仅运行本地自动预览验证。

## 验收标准

1. 四个偏好区块各显示一个与数据管理一致的灰色圆点。
2. 四个区块的标题及下方控件共享同一缩进起点，不再顶住左边缘。
3. 编辑按钮仍在右侧，点击热区不低于 `88rpx`。
4. 各区块分隔线保持完整宽度。
5. 其他五个模块的圆点、箭头、间距和行为不变。
6. Profile 相邻测试、完整测试及微信自动预览通过。

## 测试策略

- 在 `tests/profile-command-list-redesign.test.js` 增加偏好列表专属类、四个区块、圆点伪元素、缩进和完整分隔线的源码契约。
- 先运行新契约确认失败，再实现 WXML/WXSS 并确认通过。
- 运行 Profile 相邻测试与完整 committed 测试套件。
- 从真实工作区生成干净微信预览包，核对包内 WXML/WXSS 与源码一致。
