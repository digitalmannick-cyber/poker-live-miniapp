# “我的”页面控件一致性修复设计

## 背景

P5 改版后，为满足不低于 88rpx 的点击热区，筹码单位和“编辑”控件直接使用了 `min-height: 88rpx`。这把点击尺寸和视觉尺寸绑在一起，造成筹码单位、编辑按钮、预设标签之间比例失衡。数据管理使用字符 `>`，帮助与反馈等模块使用 `›`，也导致列表箭头外观不一致。

## 目标

- 统一偏好设置内三类控件的视觉比例。
- 保留直接交互控件不低于 88rpx 的可靠点击热区。
- 统一所有命令列表的箭头字形、尺寸、容器和加载状态。
- 不改变现有数据绑定、编辑弹层、导入导出、客服、AI 提醒或账号行为。

## 设计

### 筹码单位

- 新增独立的点击包装层，由包装层承载 `bindtap="selectChipUnit"` 和 `data-value`。
- 包装层最小高度为 88rpx。
- 内部可见按钮统一为 60rpx 高，四项等宽。
- 选中态继续使用红色描边和底部红色强调，不改变当前单位语义。

### 编辑按钮

- 场地预设、级别预设、对手类型三个入口使用同一点击包装层。
- 包装层最小尺寸为 88×88rpx，并由包装层承载原有编辑事件。
- 内部可见“编辑”按钮高度统一为 56rpx，宽度、内边距、字体和圆角一致。
- 预设标签继续保持紧凑胶囊形态，不扩大到 88rpx。

### 列表箭头

- 数据管理、帮助与反馈、账号与安全统一使用 Unicode 字符 `›`。
- 所有箭头继续复用 `.setting-arrow`，使用相同的 44×44rpx 圆形容器、字号、颜色和居中方式。
- 导入进行中仅把字符替换为 `…`，不改变容器尺寸和布局。

## 非目标

- 不重新设计玩家信息 Hero、模块标签、列表行、AI 开关或退出按钮。
- 不修改版本、路由、后端、客服配置和数据模型。
- 不上传微信开发版。

## 验收标准

1. 四个筹码单位可见按钮等宽且视觉高度均为 60rpx，点击包装层不低于 88rpx。
2. 三个编辑按钮的可见高度均为 56rpx，点击包装层不低于 88×88rpx。
3. 预设标签保持当前紧凑尺寸。
4. 所有非加载列表箭头均使用 `›`，不存在旧字符 `>`。
5. 数据导入加载态使用 `…`，箭头容器不跳动。
6. 所有原有点击处理器仍归属于正确控件。
7. Profile 专项测试、相邻弹层测试和完整已提交测试通过。
8. 微信开发者工具 `auto-preview` 成功，预览包包含新的包装层和统一箭头。

## 测试策略

- 源码契约测试精确断言包装层事件、视觉控件嵌套关系和尺寸选择器块。
- 断言数据管理与其他命令模块统一使用 `›`，并保留导入加载态 `…`。
- 运行 Profile 相关回归、JS 语法检查、差异检查和完整测试。
- 对真实主工作区执行微信 `auto-preview`，不执行开发版上传。
