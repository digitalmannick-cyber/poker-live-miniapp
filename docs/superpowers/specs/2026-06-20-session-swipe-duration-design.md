# Session 左滑管理与时长显示设计

## 目标

统一 Session 列表与手牌复盘列表的管理方式，并在 Session 详情页突出展示牌局时长。

## 已确认交互

### Session 列表

- Session 卡片支持水平左滑。
- 左滑后显示“编辑”和“删除”两个操作，尺寸、动画、颜色及手势阈值复用手牌复盘列表。
- 点击编辑进入现有 Session 详情页。
- 点击删除先弹出二次确认框，明确提示：删除后该 Session、该场全部手牌、手牌行动记录及对应结算记录都会永久删除，且无法恢复。
- 只有用户再次确认后才执行删除；取消不修改任何数据。
- 删除完成后关闭左滑状态并刷新列表。

### Session 时长

- 在 Session 详情页标题下方、开始时间字段上方增加居中大数字计时模块，采用用户选择的 B 方案。
- 视觉延续当前 P5 黑红风格：深色仪表卡、红色强调线、大号等宽数字、弱化的英文状态标签。
- 进行中显示“SESSION TIME”和从开始时间计算出的已进行时长。
- 已结束显示“TOTAL DURATION”和开始到结束之间的总时长。
- 格式固定为 `HH:MM`，不足两位补零；小时允许超过 23，例如 27 小时 5 分显示 `27:05`。
- 进行中的数字按分钟边界更新，不显示秒，避免无意义的高频渲染。
- 页面隐藏或卸载时停止定时器，重新显示时立即按当前时间重算，避免后台计时漂移。
- 暂停中的 Session 以 `timerPausedAt` 作为当前计时终点，数字保持不变；恢复后沿用现有开始时间平移逻辑继续计时。
- 时间字段缺失或非法时显示 `--:--`，不抛出页面错误。

## 数据与服务设计

### 时长计算

在 Session 详情页面提供独立、可测试的纯函数：

1. 解析本项目使用的 `YYYY-MM-DD HH:MM` 时间字符串。
2. 选择计时终点：进行中使用当前时间，暂停中使用 `timerPausedAt`，已结束使用 `endTime`。
3. 将非负分钟差格式化为 `HH:MM`。

该模块只负责展示，不每分钟写数据库。结束 Session 时继续由现有结束流程保存 `endTime` 和最终时长字段。

### 级联删除

为本地存储、云端仓库和 `data-service` 增加统一的 `deleteSession(sessionId)` 能力：

1. 查找目标 Session 下的全部手牌。
2. 删除这些手牌的全部行动记录。
3. 删除全部手牌。
4. 删除与 Session 关联的结算记录。
5. 最后删除 Session 本身。

本地操作保持同步一致；云端操作完成后再刷新本地业务数据。任一步失败都向页面抛出错误，页面保留当前列表并提示删除失败，避免显示“已删除”假状态。

## 页面状态

Session 列表增加单一 `swipedSessionId`，保证同一时间只有一张卡片展开。触摸状态包括起点坐标、当前 Session ID 和是否发生横向移动，用于阻止滑动结束时误触进入详情。

Session 详情增加：

- `durationDisplay`：`HH:MM` 或 `--:--`。
- `durationLabel`：进行中、暂停中或总时长对应的展示标签。
- 页面级计时器句柄，不进入响应式数据。

## 测试范围

- Session 列表具备与复盘列表一致的左滑、关闭、编辑与删除入口。
- 横向滑动不会误触详情，纵向滚动不会打开操作栏。
- 删除确认文案完整；取消时不调用删除；确认时调用级联删除并刷新。
- 本地与云端级联删除覆盖 Session、手牌、行动记录和结算记录。
- 进行中、暂停中、已结束、跨日期、超过 24 小时和非法时间的格式化结果正确。
- 页面显示时启动或刷新计时，隐藏及卸载时清理计时器。
- 现有 Session 创建、编辑、结束和 AI Session 总结流程不受影响。

## 非目标

- 不增加秒级显示。
- 不改变 Session 列表排序规则。
- 不新增恢复站或软删除功能。
- 不重做 Session 详情页其他表单区域。
