# 智牌屋启动动画设计

## 目标

为微信小程序冷启动增加一段短促、可识别的品牌动画。动画在小程序代码开始运行后、场次列表首屏显示前出现，不替代微信原生启动页。

本阶段只实现启动动画与退场衔接，不接入缓存或云端预加载。动画容器会预留完成信号，后续可以与本地初始化、轻量数据预热并行。

## 已确认主视觉

- 主视觉文件：`assets/branding/launch-phantom-five-cards-v1.jpg`
- 角色：黑色礼帽怪盗/骑士，红色轮廓光，手持恰好五张扑克牌。
- 动势：整幅人物做一次低幅度、慢节奏的上下悬浮，轮廓红光做低幅度呼吸；不再单独移动斗篷贴片。
- 品牌名称：智牌屋。
- 项目内图片为 720 × 1080、约 120 KB 的压缩版；高分辨率生成源图不进入小程序主包。

## 视觉与动效

启动层采用全屏深黑背景，主视觉以 `aspectFill` 或等效裁切覆盖屏幕。人物面部、五张牌和斗篷主体必须在常见手机比例下保持可见。

动画分为三段：

1. 进入（0–240 ms）：画面由黑色淡入，人物从 1.025 倍缩放回 1 倍，红色轮廓由弱到正常。
2. 停留（约 350–2630 ms）：整幅人物在 2.95 秒内完成一次缓慢的上下悬浮，并叠加局部红色光带。由于当前素材是单张位图，第一版不单独移动斗篷局部，避免产生贴片感。
3. 退场：红色斜切遮罩用 520 ms 从左下向右上划过，并与最后 240 ms 的启动层淡出重叠，露出已经挂载的场次列表页。

品牌名“智牌屋”位于下方安全区，使用白色高字重中文，配一个短红色斜切底块；英文标语固定为 `AI-POWERED POKER INSIGHT`。不得在图片内生成文字，避免字体失真；文字由 WXML 渲染。

## 时长规则

- 冷启动默认目标总时长：2950 ms。
- 最长硬上限：3000 ms。
- 热启动或页面返回不重复播放。
- 若系统开启减少动态效果，则取消缩放和漂移，只保留 500–700 ms 的淡入淡出。

## 页面结构

第一版在当前首个页面 `pages/session-list/session-list` 内增加全屏启动层，而不新增独立启动页路由。这样可以让真实首屏在遮罩下同步创建，退场时无需额外跳转，也不会出现导航栏和自定义 TabBar 二次闪烁。

启动层包含：

- 背景和主视觉图片；
- 红色轮廓/斜切装饰层；
- 品牌名称；
- 无障碍描述；
- 退场状态类。

页面通过一次性的进程内标记判断是否已播放。首屏卸载再进入时不重复播放；小程序进程重启后重新播放。

## 启动期间统计预热

冷启动首次播放动画时，在页面加载 600ms 后后台调用现有 `prefetchStatsData('all')`：

- 已有 `all` 统计缓存时不重复请求；
- 未登录或没有玩家 ID 时不发起请求；
- 复用 `data-service` 已有的并发去重、异常捕获和缓存失效；
- 预热失败静默处理，不展示错误，不改变启动动画时长；
- 不预加载完整手牌列表，不提前构建手牌视图。

动画严格在 2950ms 退场，绝不等待统计接口或完整云数据。

## 验收标准

1. 冷启动可见启动动画，并在 3000 ms 内完成退场。
2. 热启动、Tab 切换和从子页面返回不重复播放。
3. 画面中品牌名准确显示为“智牌屋”。
4. 手中扑克牌清晰可数且恰好五张。
5. 动画结束后场次列表无白屏、黑屏或 TabBar 跳闪。
6. 动画期间不阻塞现有 `store.initStore()` 和后台云同步。
7. 主包新增运行时图片约 143 KB，不引入视频、GIF 或第三方动画库。
8. 动画异常时最多 3000 ms 自动关闭，不阻断用户进入小程序。

## 测试与验证

- 单元测试启动状态判定：冷启动播放、同进程二次进入不播放、超时必退场。
- 静态测试 WXML/WXSS 中品牌名、图片路径、时长上限和退场类存在。
- 微信开发者工具真机比例预览，检查主视觉裁切、导航栏、TabBar 和安全区。
- 在真实工作区运行自动预览验证；预览与开发版上传分离，本阶段不自动上传。

## 非目标

- 不替换微信自身的原生启动画面。
- 不预加载完整手牌列表、玩家详情或其他非统计页面数据。
- 不使用整段视频或 GIF。
- 不单独移动斗篷贴片；整幅人物只做低幅度上下悬浮。
