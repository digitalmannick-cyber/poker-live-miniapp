# Session 历史列表 P5 输赢主视觉重设计

## 背景

当前“全部 Session”列表字段完整，但视觉层级接近普通信息卡：场地、日期、状态和 AI 总结占据主要注意力，输赢金额仅作为右下角辅助文字，用户无法快速扫视多场牌局的结果。

本次采用已确认的“Phantom Result Slash”视觉方向，并按最终反馈移除所有 Session 编号。设计使用 P5 的高对比、斜切、错位分区和少量网点纹理，但继续沿用小程序现有的黑、红、青配色和底部导航。

## 目标

- 重新设计“全部 Session”历史列表，使每场输赢成为卡片第一视觉重点。
- 保留场地与级别、日期、买入、手牌数、AI 总结、状态和输赢字段。
- 保留点击进入详情、左滑编辑/删除和 AI 总结入口的现有行为。
- 只改历史 Session 列表，不改进行中 Session、计时、速记、结算和数据模型。

## 页面结构

### 页头

- “全部 Session”使用白底黑字的 P5 斜切标题块，并保留少量红色切角装饰。
- “新建”改为右侧红色斜切按钮，文案显示为“+ 新建”。
- 不新增更新时间、筛选、排序或统计信息。

### Session 卡片

- 每条 Session 仍是独立、完整、可点击的横向卡片，保持足够触控高度。
- 卡片整体使用深黑渐变、细灰描边和轻微 P5 斜纹；正收益使用青色边缘，负收益使用红色边缘。
- 卡片不显示 Session 编号，也不保留仅服务于编号的装饰线。
- 左侧信息区约占 56%：
  - 第一行：场地与级别，例如 `MGM 200/400`。
  - 第二行：日期，例如 `2026-07-15`。
  - 底部元数据：`买入 298000`、`手牌 5`、`AI总结`、`已结束`。
- 右侧结果区约占 44%：
  - 采用向左切入的红色或青色斜切面板，与卡片融为一体。
  - 小标题为“本场输赢”。
  - 输赢金额使用当前单位格式和正负号，字号、字重和对比度均高于其他字段。
  - 正收益使用现有 `#00d1ff`，负收益使用现有 `#ff3b3b`。
- `AI总结` 保持青色描边操作；`已结束` 保持低对比灰色胶囊，不与输赢争夺视觉焦点。

## 响应式与长内容

- 以当前手机截图宽度为主基准，同时支持更窄屏幕。
- 左侧标题允许在必要时收缩或省略，但不得挤压右侧输赢金额。
- 输赢金额保持单行；超长金额通过自适应字号或减少字符间距处理，不换行。
- `AI总结` 和 `已结束` 保持最小触控尺寸；空间不足时优先收紧元数据间距，而不是隐藏字段。
- 卡片装饰层必须设置为不可交互，不能覆盖整行点击或左滑手势。

## 数据与交互

- 继续使用现有 `sessions` 列表和 `buildSessionListItem` 输出，不新增后端字段。
- 继续使用 `item.title`、`item.date`、`item.buyIn`、`item.handCount`、`item.totalProfitDisplay`、`item.summaryEligible` 和 `item.status`。
- 收益/亏损视觉状态使用现有 `item.totalProfitTone`，避免在模板中重复金额判断。
- 继续保留：
  - 点击卡片进入 Session 详情。
  - 左滑显示编辑和删除操作。
  - 点击 `AI总结` 阻止父级跳转并打开 Session 总结。
  - 已有新手引导定位 class。

## 非目标

- 不增加 Session 编号。
- 不增加筛选、排序、搜索、图表或汇总数据。
- 不修改进行中 Session 页面、弹窗、Agent、云函数、版本号和发布公告。
- 不新增人物、筹码、扑克牌等图片素材。
- 不上传微信开发版；仅在实现完成后生成本地主工作区预览。

## 验收标准

1. “全部 Session”标题与“+ 新建”使用确认稿中的 P5 斜切结构。
2. 每张历史 Session 卡片完整显示场地/级别、日期、买入、手牌数、AI 总结、状态和输赢。
3. 卡片不显示任何 Session 编号或编号占位装饰。
4. 输赢位于右侧独立斜切区域，字号与字重明显高于元数据，并保持单行。
5. 正收益为青色、负收益为红色，卡片边缘与结果区色调一致。
6. AI 总结和已结束标签尺寸统一、基线对齐，并弱于输赢视觉层级。
7. 点击详情、左滑编辑/删除、AI 总结和新建 Session 行为不变。
8. 窄屏和长金额状态下不发生遮挡、换行或点击区域丢失。
9. Session 列表专项测试和完整测试通过。
10. 微信开发者工具在真实主工作区完成 `auto-preview`，预览包与源文件一致。

## 测试策略

- 新增 Session 列表视觉契约测试，断言 P5 页头、卡片左右分区、结果色调、无编号和单行金额规则。
- 复用现有 Session 列表交互测试，确认详情跳转、左滑操作和 AI 总结入口未被结构调整破坏。
- 运行 Session 专项测试、完整测试套件和微信 `auto-preview`。
- 将预览结果与最终确认图进行人工对照，重点检查斜切比例、输赢层级、元数据基线和底部导航遮挡。
