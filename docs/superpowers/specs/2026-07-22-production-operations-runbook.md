# 智牌屋正式上线值守与恢复手册

## 1. 适用基线

- 小程序业务版本：`3.38`
- 微信开发版本：`1.0.20260722.2243.v3.38`
- 代码/云端上线基线：`v3.38-launch-ready.2`
- CloudBase 环境：`cloud1-d3ggy9aq3be912e34`
- 正式发布前环境：邀请小程序码 `trial`，AI 提醒 `trial`
- 正式发布后环境：邀请小程序码 `release`，AI 提醒 `formal`

本手册不授权提交审核、正式发布、覆盖生产集合、删除恢复演练集合或修改隐私/安全配置。上述动作必须由发布负责人确认。

## 2. 发布角色

正式提交审核前必须填写：

| 角色 | 姓名/账号 | 职责 |
| --- | --- | --- |
| 发布负责人 | 待填写 | 确认版本、提交审核、正式发布 |
| 回滚负责人 | 待填写 | 决定回滚并执行云函数/小程序恢复 |
| 数据负责人 | 待填写 | 核对备份、恢复演练和数据完整性 |
| 内容管理员 | 当前管理员账号，具体账号不写入 Git | 删除违规公开评论，保留处置记录 |
| 上线值守 | 待填写 | 前 24 小时监控、记录并升级故障 |

没有发布负责人和回滚负责人时，结论为 `NO-GO`。

## 3. 上线监控阈值

| 指标 | 告警阈值 | 立即动作 |
| --- | --- | --- |
| `poker_data` / `poker_social` 5 分钟错误率 | `> 5%` | 停止发布或新功能宣传；核对最近部署与 CloudBase 状态 |
| 登录或社交资料同步失败 | 连续 5 分钟或 3 个不同真实账号复现 | 停止发布；不得用本地缓存“假成功”掩盖 |
| 广场读取失败 | 5 分钟失败率 `> 3%` | 临时关闭广场入口或回滚 `poker_social` |
| 评论/点赞写入失败 | 5 分钟失败率 `> 5%` | 暂停互动入口，保留牌谱浏览 |
| 评论安全检测异常 | 任一违规内容被明确放行，或检测服务连续异常 5 分钟 | 暂停广场文字评论；管理员处置已有内容 |
| 私有数据越权 | 任意一次 | 立即停止公开入口并回滚，不等待统计显著性 |
| BB 化/匿名化失效 | 任意一次 | 立即撤下对应分享并停止广场发布 |
| AI 复盘错误/超时 | 15 分钟错误率 `> 10%` | 只降级 AI 入口，不阻塞记录、统计和社交 |
| 云资源消耗 | 单小时达到日预算的 `30%` | 限制高频入口，检查异常调用和限流 |

上线前 2 小时每 15 分钟记录一次；2—24 小时每 2 小时记录一次；24—72 小时每天至少两次。

## 4. 正式发布顺序

1. 确认微信后台审核通过的版本、构建号和 Git 标签一致。
2. 由发布负责人点击正式发布。
3. 记录发布时刻、微信版本号、Git 提交和操作人。
4. 执行：

   ```powershell
   .\tools\production-environment-switch.ps1 `
     -Apply `
     -ConfirmPublished `
     -ConfirmEnvironmentId cloud1-d3ggy9aq3be912e34 `
     -ExpectedCommit <发布提交> `
     -ConfirmVersion 3.38
   ```

5. 再运行一次不带参数的脚本，必须显示 `invite=release; reminder=formal`。
6. 用非体验成员微信扫码进入邀请页，并触发一次 AI 提醒验证正式版跳转。

发布前不得提前执行第 4 步。

## 5. 数据回档与隔离恢复演练

2026-07-22 的只读检查确认：

- 可回档区间：`2026-07-14 03:55:19` 至 `2026-07-22 22:57:18`
- 检查点 `2026-07-22 04:12:20` 包含当前 29 个业务集合

每次上线前重新查询，不能把上述时间当成永久状态：

```powershell
tcb db nosql backup time --json
tcb db nosql backup collection --time '<检查点时间>' --json
```

恢复演练必须满足：

1. 只选择一至两个无敏感展示需求的代表集合。
2. 使用 `OldTableName`/`NewTableName` 映射到 `recovery_drill_<日期>_<原集合>`。
3. 禁止把 `NewTableName` 设置为任何现有生产集合名。
4. 回读文档数量和关键字段，记录演练结果。
5. 演练副本包含真实用户数据，禁止导出、截图或共享。
6. 清理演练集合属于数据删除，必须由数据负责人明确确认后单独执行。

示例仅用于评审，不得直接复制执行：

```powershell
tcb db nosql backup restore `
  --time '<检查点时间>' `
  --tables '[{"OldTableName":"sessions","NewTableName":"recovery_drill_YYYYMMDD_sessions"}]' `
  --json
```

## 6. 回滚路径

### 6.1 仅社交或评论故障

1. 暂停广场/评论入口，保留本地记牌与统计。
2. 如管理员配置异常，使用现有 fail-close 路径停用管理员能力。
3. 恢复 `poker_social` 的上一稳定代码；保留集合与索引，不做数据库降级删除。

### 6.2 核心数据故障

1. 立即停止新增写入入口和正式宣传。
2. 固化故障时刻、受影响账号范围、请求 ID 和当前 Git/云函数版本。
3. 恢复上一稳定代码 `2858271` 或经负责人确认的后续稳定标签。
4. 数据回档先恢复到隔离集合比对；未经数据负责人确认，禁止覆盖生产集合。

### 6.3 正式环境跳转配置错误

1. 发布后若小程序码或订阅消息仍指向体验版，先运行只读环境检查。
2. 仅在正式版本确已发布时重新执行双环境切换工具。
3. 若正式版本撤回，需把邀请环境恢复为 `trial`、提醒环境恢复为 `trial`；保留其他环境变量。
