# 智牌屋上线隐私声明与数据处理清单

## 1. 使用方式

本文根据 `3.39` 候选源码逐项整理，用于填写微信公众平台“用户隐私保护指引”和审核备注。后台的实际字段名称与可选用途以提交时微信公众平台为准；不能把本文当成已经在后台完成配置的证据。

## 2. 必须披露的数据与能力

| 信息或能力 | 触发方式 | 用途 | 保存及可见范围 | 删除方式 |
| --- | --- | --- | --- | --- |
| 微信 OpenID | 用户进入小程序并使用云端功能 | 识别同一微信用户、隔离云端数据、校验好友与内容权限 | 仅云函数和受限数据库内部使用，不返回客户端页面或其他用户 | “清除所有数据”时删除或去标识化业务记录；必要的操作元数据仅保留无业务载荷版本 |
| 昵称 | 用户在原生昵称输入控件确认 | 建立公开社交资料、好友搜索、动态与评论署名 | 使用好友/广场功能后对其他已登录用户可见；变更时经过微信资料场景安全检测 | 可在“我的”修改；清除所有数据后替换为“已注销用户” |
| 头像 | 用户点击原生头像选择或主动从相册选择 | 个人资料、好友列表、动态与评论头像展示 | 用户选择同步后上传微信云存储；社交页面仅返回临时访问地址 | 可更换或移除；清除所有数据时删除本应用管理的云存储头像文件 |
| 相册图片/拍照结果 | 用户在玩家详情主动选择图片 | 作为本地玩家库头像或好友名片头像 | 保存于本地及微信云存储；默认仅用户自己可见，主动分享玩家名片时向一名指定好友展示副本 | 删除/更换记录后引用消失；清除所有数据时删除本应用管理的玩家头像文件 |
| 场次、手牌及统计 | 用户主动创建、记录或导入 | 牌谱记录、统计、Replay 和 AI 复盘 | 原始数据仅本人可见；只有主动发布的 BB 化、非 Hero 匿名快照按所选范围展示 | 可逐项删除；“清除所有数据”删除云端与本地业务数据 |
| 玩家库 Note、Leak、类型及对战手牌 | 用户主动填写 | 个人牌手观察与复盘 | 默认仅本人可见；主动分享一张玩家名片时，仅向一名指定好友展示当时副本 | 可编辑/删除；“清除所有数据”统一删除 |
| 广场评论与回复 | 用户主动提交 | 牌谱讨论 | 广场文字对所有已登录用户可见；提交前经过微信评论场景安全检测 | 用户可删除自己的评论；管理员可软删除违规评论；清除账号时作者身份去标识化 |
| 剪贴板 | 用户点击复制、导入或粘贴按钮 | 导出牌谱/备份，或读取用户主动粘贴的备份内容 | 复制内容写入系统剪贴板；读取内容仅在用户主动导入后处理，确认导入的数据按正常云同步规则保存 | 用户可覆盖系统剪贴板；已导入内容按对应业务数据删除 |
| 微信消息文件 | 用户点击“从文件导入” | 导入用户选择的备份文件 | 文件在本机解析，确认后的业务数据按正常云同步规则保存 | 按业务数据删除或“清除所有数据”统一删除 |
| 订阅消息授权 | 用户主动开启 AI 提醒 | 发送用户选择的牌谱复盘提醒 | 只记录提醒设置与微信返回的授权结果，不收集手机号 | 可关闭提醒；清除所有数据时删除设置 |
| AI 复盘输入 | 用户主动请求 AI 建议、问答或场次总结 | 生成牌谱复盘、策略回答与总结 | 向自有 AI Agent 服务发送所需的牌谱上下文、问题和稳定匿名用户 ID；不发送原始微信 OpenID | 随对应牌谱/会话数据删除；服务端日志不得记录原始 OpenID 或完整私有载荷 |
| 客服会话 | 用户点击“反馈与建议” | 接收问题、Bug 和功能建议 | 由微信原生客服会话承载，应用自己的数据库不保存客服消息 | 用户和管理员按微信客服能力管理会话 |

## 3. 第三方与云端处理方

公众平台隐私指引至少应与以下实际链路一致：

1. **微信/腾讯云开发**：登录身份、云数据库、云存储、云函数、订阅消息、小程序码与内容安全检测。
2. **自有 AI Agent 服务**：接收用户主动发起复盘时所需的牌谱上下文和问题；用户标识在云函数内先转换为不可逆的稳定匿名 ID。
3. **微信原生客服**：用户主动进入客服会话后承载文字和图片反馈。

当前正式候选界面不申请麦克风权限，也不直接录制或上传音频；“语音输入”使用手机输入法完成。代码中的音频转写预留能力不得在未更新隐私指引、权限声明和删除策略前开放。

## 4. 公开与私有边界

- 永远私有：原始盈利/金额、场所、完整场次、未发布手牌、Note、Leak、本地好友备注、真实非 Hero 玩家名、OpenID。
- 好友可见：用户授权的累计时长、记录手牌数、称号，以及明确发布到好友范围的匿名 BB 手牌。
- 广场可见：用户主动发布的匿名 BB 手牌、公开昵称/头像、点赞数及广场评论。
- 指定好友可见：定向手牌和一张指定玩家名片的快照；不能由接收方继续读取发送方玩家库。
- 微信页面索引：当前没有独立的公开落地页，`sitemap.json` 对全部业务页面设置 `disallow`。这只关闭页面内容索引，不改变用户按小程序名称搜索和正常进入小程序的能力。

## 5. 公众平台建议填写口径

2026-07-23 在当前账号的“增加信息类型”选择器中只读实查到可直接选择“用户信息（微信昵称、头像）”“发布内容”“操作日志”和“自定义”等类型。按 `3.39` 的真实实现，建议映射如下：

| 后台信息类型 | 本版本是否需要 | 建议用途 |
| --- | --- | --- |
| 用户信息（微信昵称、头像） | 是 | 用户主动建立公开资料、好友搜索以及动态和评论署名 |
| 发布内容 | 是 | 保存用户主动发布的匿名 BB 牌谱、广场评论和回复，并按所选范围展示 |
| 操作日志 | 是 | 保障账号安全、写入幂等、限流和管理员评论处置审计；不记录完整私有牌谱载荷 |
| 自定义：账号标识与云端牌谱/训练数据 | 是 | 使用微信 OpenID 隔离本人数据，保存用户主动记录的场次、手牌、统计、玩家资料、好友关系及提醒配置 |
| 自定义：AI 复盘输入 | 是 | 用户主动请求复盘时发送必要牌谱上下文、问题和不可逆匿名用户标识到自有 AI Agent 服务 |
| 麦克风 | 否 | 候选运行时代码未调用录音接口；应从当前后台声明中移除，不把手机输入法语音输入当作小程序采集麦克风 |

相册/视频、选中文件和剪贴板已经由平台根据候选代码识别，仍需保留与真实按钮触发方式一致的用途说明。不要勾选定位、地址、手机号、摄像头等未使用类型。

可在用途描述中使用以下表达，并按后台实际选项拆分：

- “用于用户主动创建个人资料、好友搜索及牌谱动态署名。”
- “用于用户主动选择玩家头像或个人头像，并在其授权的资料、好友或分享范围内展示。”
- “用于用户主动导入或导出牌谱备份，不在后台静默读取剪贴板。”
- “用于保存用户主动记录的牌谱，并生成个人统计与复盘结果。”
- “用于用户主动请求 AI 牌谱复盘；仅传输完成本次分析所需的数据，并使用匿名用户标识。”

不要填写或暗示：定位、通讯录、手机号、身份证、银行卡、交易结算、后台持续读取剪贴板、后台录音。当前版本没有这些能力。

`app.json` 当前有意不配置 `requiredPrivateInfos`：微信开发者工具当前配置说明将该字段定义为地理位置相关隐私接口声明，而本版本不调用定位接口。不得为了“补全配置”添加空数组或无实际用途的定位声明；未来一旦引入定位、录音、手机号、地址、发票或微信运动数据，必须先更新隐私指引、交互授权、删除策略和自动化测试，再提交新版本。
