# 版本更新海报设计

## 目标

每次发布新版本后，已登录用户至少确认一次本版本的更新内容。公告以海报弹层展示，不提供关闭按钮，只有点击“我知道了”后才能关闭。同一账号在同一版本确认后不再自动弹出，但可以在“我的”页面主动重新查看。

## 非目标

- 不建设通用运营公告系统。
- 不建设管理员后台、定时发布、用户分组或公告统计。
- 不从 CloudBase 动态修改版本说明，避免说明内容与已发布代码版本不一致。

## 内容来源

新增 `config/release-notes.js`，以 `config/app-version.js` 的 `displayVersion` 为当前发布版本来源。配置包含：

- `version`：必须与 `displayVersion` 一致。
- `eyebrow`：短标签，例如“VERSION UPDATE”。
- `title`：本次更新标题。
- `summary`：一句话说明。
- `items`：更新项目列表，每项包含标题和可选说明。
- `imageUrl`：可选云存储文件 ID 或 HTTPS 图片地址。
- `acknowledgeText`：默认“我知道了”。

配图不放入本地资源包。发布前将图片上传到云存储，再把地址写入版本配置，以控制小程序包体积。没有配图或图片加载失败时，海报自动使用纯文字布局。

自动化测试必须校验 `releaseNotes.version === appVersion.displayVersion`，防止版本号递增后忘记更新说明。

## 展示规则

确认状态使用账号隔离的本地存储键：

`releaseNotesAck:<playerId>:<version>`

展示条件同时满足：

1. 用户已经登录，不是退出登录状态。
2. 当前版本存在有效更新说明。
3. 当前账号尚未确认该版本。
4. 当前页面不存在登录授权、资料同步等更高优先级阻塞弹层。

正常冷启动时，在 Session 主页面完成首屏数据加载后检查并展示；用户刚在“我的”完成微信登录时，在登录流程结束后检查并展示。两处使用同一个状态模块和同一个海报组件，避免重复逻辑。

若用户通过深链进入其他业务页，不强制中断当前操作；回到 Session 或“我的”主页面时再展示。

## 交互

- 海报为居中大幅弹层，背景遮罩禁止点击关闭。
- 不显示右上角关闭按钮。
- 系统返回操作不关闭海报。
- 唯一操作为底部“我知道了”。
- 点击后先写入确认状态，写入成功后关闭。
- 写入失败时保持海报可见，并提示“确认失败，请重试”。
- “我的”页面增加“版本更新”入口，点击后可随时打开当前版本海报；主动查看不会清除或改变已确认状态。

## 视觉结构

沿用当前深色、红色和青色的 P5 视觉语言，但保证正文可读性：

1. 顶部版本标签与标题。
2. 可选横幅配图。
3. 一句话摘要。
4. 3～6 条更新内容，超出时海报内容区内部滚动。
5. 底部固定“我知道了”按钮。

弹层宽度适配手机安全区，正文不使用图片内文字作为唯一信息来源。配图只是辅助，所有关键更新必须同时存在于结构化文字中。

## 模块边界

- `config/release-notes.js`：只负责当前版本内容。
- `utils/release-notes.js`：负责账号级确认键、是否应展示、确认写入。
- `components/release-notes-poster/`：只负责展示和发出确认事件。
- Session 页面：首屏稳定后触发展示。
- “我的”页面：登录完成后触发展示，并提供手动入口。

组件不直接读写业务数据；页面通过状态模块得到展示结果并处理确认，便于单元测试。

## 异常与优先级

- 配图失败：隐藏配图，继续展示文字。
- 配置缺失或版本不匹配：不弹出并记录诊断日志；测试阶段直接失败。
- 未登录：不展示，不写匿名确认状态。
- 登录资料同步弹层仍在显示：版本海报排队，待同步弹层结束后再出现。
- 本地存储被清理：该版本会再次展示，可接受，因为仍满足“至少确认一次”的目标。

## 验收标准

1. 已登录且未确认当前版本的用户进入小程序后看到更新海报。
2. 海报没有关闭按钮，点击遮罩和返回操作不能关闭。
3. 点击“我知道了”后关闭；同一账号再次进入不再自动展示。
4. 换账号后，新账号仍需确认当前版本。
5. `displayVersion` 变化后，原账号必须重新确认新版本。
6. 用户可从“我的 > 版本更新”重新打开当前版本内容。
7. 图片缺失或加载失败不影响文字和确认按钮。
8. 登录资料同步弹层与版本海报不会重叠。
9. Session 主页面和“我的”页面复用同一个组件及确认规则。

## 测试

- 状态单测：未登录、未确认、已确认、换账号、换版本、存储失败。
- 配置单测：版本号必须与 `displayVersion` 一致，更新条目不能为空。
- 组件测试：无关闭入口、遮罩不可关闭、确认事件、图片失败降级。
- 页面流程测试：冷启动展示、登录后展示、资料同步完成后展示、“我的”手动重开。
- 真机预览：常见手机宽度、安全区、长内容滚动、无图和有图两种布局。
