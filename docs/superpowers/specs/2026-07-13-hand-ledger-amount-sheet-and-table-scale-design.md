# 手牌完整录入金额面板与桌面空间优化设计

## 目标

1. 将 Raise/Bet 与玩家有效筹码的数字录入统一为一个可复用的小程序组件。
2. 有效筹码每次打开从 0 开始，并提供 100bb、200bb、300bb 快捷值。
3. 继续扩大完整录入桌面、玩家座位及座位内手牌，减少底部无效留白。
4. 让当前行动座位直接呈现清晰、持续但不过度干扰的动态高亮。

## 金额面板组件

新增 `components/numeric-amount-sheet`，由组件维护尚未提交的数字草稿，父页面维护业务状态。

### 输入属性

- `visible`：是否展示。
- `title`：面板标题，例如 `Raise` 或 `设置 SB 有效筹码`。
- `value`：打开时的金额。
- `unit`：筹码单位，例如 `HK$`。
- `presets`：快捷项数组，每项包含 `label` 与数值 `value`。
- `max`：滑杆及金额校验上限。
- `accent`：主题色，Raise/Bet 使用黄色，有效筹码使用黄色筹码主题。
- `secondaryLabel`：可选次要命令；有效筹码传入 `其他玩家设置同样筹码`。

### 输出事件

- `confirm`：返回已确认的正整数金额。
- `close`：关闭且不修改父页面数据。
- `secondary`：返回当前草稿金额，由父页面执行批量设置。

### 统一交互

- 数字键盘包含 0-9、清空、退格。
- 输入显示区不是系统输入框，不拉起原生键盘。
- 当草稿为 `0` 时输入数字，直接替换 0；其他情况追加数字。
- 快捷项直接替换草稿金额，并呈现选中状态。
- 滑杆与数字草稿双向同步。
- 金额为 0 或超过上限时不保存，并给出轻量提示。
- 关闭面板不写回业务状态。

## 有效筹码规则

- 每次打开玩家有效筹码面板时草稿值为 0，可直接从数字键盘输入新值。
- 已保存筹码继续显示在座位上；打开编辑面板不把旧值预填到新草稿中。
- 快捷项根据该手牌记录时的级别计算：100bb、200bb、300bb。
- 上限至少覆盖 300bb；若已有金额更大，上限扩展到已有金额。
- `其他玩家设置同样筹码` 保留为次要命令，使用当前草稿值批量更新非 Hero 玩家并关闭面板。
- 保存单个玩家只更新当前座位。

## Raise/Bet 迁移

- 保留现有翻牌前 BB 倍数与翻牌后底池比例快捷项。
- 保留现有最大可投入筹码、动作提交、历史节点及筹码动画逻辑。
- 仅将现有内联 WXML、数字键盘方法和滑杆展示迁移到复用组件，不改变扑克业务语义。

## 桌面空间与尺寸

- 页面底部预留高度改为与实际固定操作区一致，消除时间轴与操作区之间多余空白。
- 增加桌面可用高度后，椭圆桌向下自然延伸，桌框高度接近桌面容器上下边界。
- 6max、8max、9max 继续使用独立稳定坐标，不重新引入运行时椭圆公式。
- 座位尺寸整体增加约 8%-10%；9max 使用受控的紧凑尺寸，优先保证不重叠。
- 座位内手牌增加约 10%，仍完整包含在座位主体内。
- 下注金额继续使用独立桌内锚点，不跟随座位放大，确保不遮挡公共牌与底池。
- 公共牌、底池与阶段提示保留中央安全区。

## 当前行动状态

- Hero 黄色边框表示身份，永久保留。
- 当前行动增加独立的青绿色外环，不覆盖 Hero 边框。
- 外环使用缓慢呼吸动画，并带轻微座位缩放；折叠玩家不显示行动高亮。
- 删除独立漂浮的 `turn-flow` 小圆点，避免两个视觉焦点同时表达同一状态。
- 动画仅作用于 `transform`、`opacity` 与阴影，不触发布局位移。

## 测试与验收

### 自动化测试

- 金额组件首次展示、快捷替换、0 开始输入、清空、退格、滑杆同步、确认与关闭。
- 有效筹码每次打开默认 0，数字输入行为与 Bet/Raise 一致。
- 100bb、200bb、300bb 按手牌大盲正确换算。
- 单人保存与批量同步分别更新正确玩家。
- Raise/Bet 迁移后现有金额与动作测试继续通过。
- WXML 中不再保留两套重复数字键盘。
- 6/8/9max 座位边界、相邻距离、下注锚点和中央安全区继续通过。

### 视觉验收

- 常见手机视口下桌面下方不再出现明显无效空白。
- 6/8/9max 座位、手牌、玩家信息、下注金额互不遮挡。
- 当前行动座位在一秒内可识别；Hero 当前行动时黄色身份边框与青绿色行动外环同时可见。
- 桌面区域无滚动条，底部操作按钮无遮挡。

## 不在本次范围

- 不改变行动顺序、底池计算、有效筹码计算或保存数据结构。
- 不改变快速记牌页面的金额录入。
- 不新增前端牌谱 parser、normalizer 或本地语义兜底。
