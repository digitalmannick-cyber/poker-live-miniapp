# 我的页面统计首屏稳定设计

## 问题

“我的”页面在首次进入时先显示硬编码的 `0.0` 或本地旧 Session 计算出的累计小时数，随后等待云端 `sync_stats` 返回再覆盖，导致累计小时小数闪动。当前统计缓存只存在进程内存中，重新编译或冷启动后无法复用上次已经确认的云端结果。

## 设计

- 只持久化 `all` 范围的统计摘要，不持久化 Sessions、Hands 或分析图表数据。
- 摘要必须携带当前 `playerId`；读取时账号不一致即视为无缓存，防止串号。
- 云端统计成功返回后写入摘要；本地业务数据发生变更、退出或切换账号时清除当前账号摘要。
- Profile 页面模块初始化时同步读取摘要，用它生成 `stats` 和 `titleProgress`，避免先渲染硬编码 `0.0`。
- 没有可信摘要时显示尺寸稳定的轻量骨架，等首次统计返回后直接显示真实数字，不显示临时小数。

## 验收标准

- 同一账号冷启动后，已缓存的累计小时首帧即为最近一次可信云端值。
- 云端刷新完成且数字未变化时，不产生二次统计区域渲染。
- 缓存属于其他账号时不可读取。
- 本地业务数据变更或账号状态变化后，不继续使用旧摘要。
- 没有摘要时不显示 `0.0` 作为临时累计小时。
- 版本号递增，并完成微信开发者工具自动预览；不上传开发版。

## 测试

- Node 回归测试覆盖摘要写入、账号隔离、清除和 Profile 首屏初始化。
- 运行现有统计缓存与 Profile 布局测试。
- 运行 JavaScript 语法检查和微信开发者工具自动预览。
